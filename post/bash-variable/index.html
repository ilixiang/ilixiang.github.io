bash - 变量及变量操作

Friday, June 12, 2020

基本变量操作

变量声明

bash中的变量有2个特点：

  1. 变量不存在类型
  2. 变量的作用域默认为全局

这两个特点决定了bash中的变量可以在你想要的任何时候定义，并且不会因为定义在循环或者条件分支中而失效。比较特殊的例子就是数组类型：

#!/bin/bash
# test.sh

arr[0]="hello"
arr[1]="world"

echo ${arr[0]}
echo ${arr[1]}

#------------------------
$ ./test.sh
hello
world

你可以不用声明这是一个数组，然后直接往里面塞东西，十分方便。

即使如此，bash依然提供declare和typeset两个内建命令去声明变量的类型，当你声明了变量的类型之后，对这个变量的赋值就会被限制，举个栗子：

#!/bin/bash
# test.sh

# declare an integer variable
declare -i arr=3

# assign string to varialbe
arr="hello"

echo $arr

#-------------------
$ ./test.sh
0

由此可见赋值失败了，比较失望的是没有报错。

对于变量的声明declare和typeset几乎拥有一样的参数，以下列举几个比较常用的参数：

  1. -i整型变量
  2. -a数组变量
  3. -r只读变量
  4. -x环境变量，等同于export
  5. -p查看变量声明的属性

数组操作

数组操作主要是如何在循环中遍历数组，主要有3个办法：

  1. 使用${arr[@]}获取数组中的变量，但由于IFS默认为空白，所以当数组元素的字符串中含有空格时会被分割为2个字符串变量，通过修改IFS='\n'可以修正，但单个元素的字符串中就不能出现\n换行
  2. 使用${#arr[@]}获取数组长度进行遍历，但bash的数组可能存在元素为空的位置
  3. 使用${!arr[@]}获取数组索引序列，不会遇到以上问题，推荐
#!/bin/bash
# test.sh

for i in $(seq 6); do
	arr[$(( i - 1 ))]=$i
done

echo "----------\${arr[@]}--------"
for str in ${arr[@]}; do
	echo $str
done

echo "----------\${#arr[@]}--------"
i=0
while [[ $i < ${#arr[@]} ]]; do
	echo ${arr[$i]}
	(( ++i ))
done

echo "----------\${!arr[@]}--------"
for i in ${!arr[@]}; do
	echo ${arr[$i]}
done

# -----------------------------------------------
$ ./test.sh 
----------${arr[@]}--------
1
2
3
4
5
6
----------${#arr[@]}--------
1
2
3
4
5
6
----------${!arr[@]}--------
1
2
3
4
5
6

算数运算

bash中的变量在大多数情况下是无法和C语言一样直接进行运算的，无论是算数运算还是逻辑运算都要使用相应的语法或者内建命令。

bash中变量在大多数情况下是无法和C语言一样直接进行运算的，无论是算数运算还是逻辑运算都要使用相应的语法或者内建命令。

常见的执行算数运算的方法有expr、let、$((expression))、$[expression]这4种，expr属于外部命令，let、$((expression))和$[expression]属于内建的算数拓展，其中$[expression]这种已被声明后续版本会移除而不建议使用。举个栗子：

#!/bin/bash
# test.sh

a=5
b=4

str="$a + $b"

echo "-----------expr----------"
echo "$a * $b = $(expr $a \* $b)"
echo "$a % $b = $(expr $a % $b)"
echo "$a + $b = $(expr $a + $b)"
echo "$a + $b = $(expr $str)"

echo "-----------let-----------"
let c=a*b d=a%b f=a+b g=str
echo "$a * $b = $c"
echo "$a % $b = $d"
echo "$a + $b = $f"
echo "$a + $b = $g"

echo "----------\$(())---------"
echo "$a * $b = $((a*b))"
echo "$a % $b = $((a%b))"
echo "$a + $b = $((a+b))"
echo "$a + $b = $((str))"

echo "----------(())-----------"
((c=a-b,d=a/b,f=a**b,g=str))
echo "$a - $b = $c"
echo "$a / $b = $d"
echo "$a ** $b = $f"
echo "$a + $b = $g"

#------------------------------
-----------expr----------
5 * 4 = 20
5 % 4 = 1
5 + 4 = 9
5 + 4 = 9
-----------let-----------
5 * 4 = 20
5 % 4 = 1
5 + 4 = 9
5 + 4 = 9
----------$(())---------
5 * 4 = 20
5 % 4 = 1
5 + 4 = 9
5 + 4 = 9
----------(())-----------
5 - 4 = 1
5 / 4 = 1
5 ** 4 = 625
5 + 4 = 9

就以上的例子而言，我觉得最好用的算数计算方法是$(())

  1. expr的缺点在于构造表达式的时候必须读取变量，并且在运算符和变量之间必须有空白，并且它支持的运算是最少的，且使用乘法需要对*进行转义
  2. let相当于(())，他执行的表达式大多需要进行赋值操作，并且它能做到的$(())都能做到
  3. $(())功能上几乎无可挑剔，拥有前面两者的全部功能，并且定义在POSIX标准中，可移植性最好

逻辑运算

bash中提供的逻辑运算方法有test、[ expression ]以及[[ expression ]]三种。前两者属于内建方式，定义在POSIX标准当中，而第三种是bash中的关键字，可移植性较低，但却是最安全的。

$ str1="a"
$ str2="b"
$ [ $str1 = $str2 ] || echo "false"
false
$ unset str1
$ [ $str1 = $str2 ] || echo "false"
-bash: [: =: unary operator expected
false
$ [[ $str1 = $str2 ]] || echo "false"
false
$ str1="a bcd"
$ [ $str1 = $str2 ] || echo "false"
-bash: [: too many arguments
false
$ [ "$str1" = "$str2" ] || echo "false"
false
$ [[ $str1 = $str2 ]] || echo "false"
false

上述示例说明了为什么使用[ expression ]是不安全的：

  1. 当变量不存在时，会报错
  2. 当字符串中含有空白时，必须使用双引号抱住才能保证正常运行

除此之外，[[ expression ]]还有支持=使用通配符，<和>字典顺序的大小比较以及=~正则表达式匹配

$ [[ abc = a* ]] && echo "true"
true
$ [[ abc < bc ]] && echo "true"
true
$ [[ abc =~ a.* ]] && echo "true"
true

因此，在bash环境下建议使用[[ expression ]]的方式进行逻辑运算

字符串操作

bash的字符串操作方法大致分为2种：

  1. 使用内建的字符串操作语法
  2. 使用expr命令进行操作
  3. 使用其他命令进行字符串操作，比如awk的内建函数或者sed的s命令等

而内建的字符串操作必须将字符串存储于变量中才能够使用，因而不适合在交互式的命令行下直接使用。而expr没有这个限制，但能够进行的操作少一些。其次，内建的字符串操作不支持正则表达式，而expr是支持的。

内建字符串操作

${#str} 获取$str的长度

$ str="0123456789"
$ echo ${#str}
10

${str:index} 从index开始截取$str中的子串至最后。

${str:index:length} 从index开始截取$str中长度为length的字串

上面的两条语句index皆可以是负数，用于表示字符串从后向前的索引

$ str="0123456789"
$ echo ${str:4}
456789
$ echo ${str:4:3}
456
$ echo ${str: -2} # 负数这里:和-之间必须有一个空格
89
$ echo ${str:(-2)} # 也可以用()将这个数括起来
89
$ echo ${str: -2:1}
8

${str#$substr}从$str中从左向右移除匹配$substr的最短的部分

${str##$substr}从$str中从左向右移除匹配$substr的最长的部分

${str%$substr}从$str中从右向左移除匹配$substr的最短的部分

${str%%$substr}从$str中从右向左移除匹配$substr的最长的部分

有点绕，要点有四：

  1. 移除方向。#代表从左向右移除，%代表从右向左移除
  2. 移除长度。#代表能少则少，## 代表能多则多；%代表能少则少，%%代表能多则多
  3. 移除什么。$substr部分可以直接换成字符串，比如从左至右移除字符串0123，可以写成${str#0123}；移除的部分可以使用通配符，这也是为什么有最长和最短匹配的原因
  4. 移除起点。必须从第一个字符开始移除，否则不移除
$ str="abcABC123ABCabc"
$ echo ${str#a*c}
ABC123ABCabc
$ echo ${str#a*A}
BC123ABCabc
$ echo ${str##a*A}
BCabc
$ echo ${str%A*c}
abcABC123
$ echo ${str%%A*c}
abc
$ echo ${str#b} # 第一个字符不匹配，不移除
abcABC123ABCabc

${str/$substr/$replacement}替换第1个与$substr匹配的部分为$replacement

${str//$substr/$replacement}替换所有与$substr匹配的部分为$replacement

${str/#$substr/$replacement}如果$substr能从头开始匹配$str的一个字串，则将之替换为$replacement

${str/%$substr/$replacement}如果$substr能从尾开始匹配$str的一个字串，则将之替换为$replacement

同样，这部分的$substr也支持使用通配符，并且能将$substr和$replacement部分直接使用字符串替换

$ str="abcABC123ABCabc"
$ echo ${str/abc/xyz}
xyzABC123ABCabc
$ echo ${str//abc/xyz}
xyzABC123ABCxyz
$ echo ${str/#abc/xyz}
xyzABC123ABCabc
$ echo ${str/#ABC/XYZ} # 无法从头匹配，不替换
abcABC123ABCabc
$ echo ${str/%abc/xyz}
abcABC123ABCxyz
$ echo ${str/%ABC/XYZ} # 无法从尾匹配，不替换
abcABC123ABCabc

expr字符串操作

expr length $str 获取$str的长度

$ str="abcABC123ABCabc"
$ expr length $str
15

expr $str : $regex 获取$str中匹配正则表达式$regex的长度

expr match $str $regex 获取$str中匹配正则表达式$regex的长度

$ str="abcABC123ABCabc"
$ expr $str : a.*A
10
$ expr match $str a.*A
10

expr substr $str $position $length 获取$str中从$position开始长度为$length的字串，$str中的第一个字符对应的$position为1而不是0

$ str="abcABC123ABCabc"
$ expr substr $str 4 6
ABC123

expr index $str $chars 获取$str中首次匹配字符集$chars中任意字符的位置索引

$ str="abcABC123ABCabc"
$ expr index $str kaib
1
$ expr index $str kib
2

字符串操作基本能够满足日常使用，但无论是内建表达式或者expr都没有能够支持正则表达式的字符串替换功能，想要做到这点可以使用sed的s命令去做，感兴趣的小伙伴可以查阅相关文档去了解一波。
