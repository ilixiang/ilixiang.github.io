Java - 枚举

Tuesday, July 14, 2020

枚举，是Java中一中比较特殊的类型。我个人而言，它是一种有穷集合，因而它的成员都是它自己的某一个元素。枚举的关键字是enum，并且在各种语言中枚举的行为并不相同。在C中，枚举的元素对应一个整型数字，虽然其变量能够作为左值，但却并不安全，能够被任意的其他枚举元素以及整型赋值。

#include <stdio.h>

// 季节枚举
typedef enum {
	SPRING,
	SUMMER,
	AUTUMN,
	WINTER
} EnumSeason;

// 颜色枚举
typedef enum {
	RED,
	BLUE
} EnumColor;

int main() {
    EnumSeason known_season = SPRING;
    // 使用100赋值
	EnumSeason unknown_season = 100;
    // 使用color_enum的元素赋值
	EnumSeason color_season = BLUE; 
	printf("%d\n%d\n%d\n", known_season, unknown_season, color_season);
	return 0;
}
// ----- 输出 ------
0
100
1

Java中的枚举类型与C不同，Java为其提供了几个比较方便的特性：

  • 编译时检查。能够避免上述的安全性问题，不同的enum之间无法赋值，整型和enum之间无法赋值
  • 特殊的toString()。enum类型的toString()方法的结果默认为该枚举的元素名称，而并非整型数字
  • values()获取元素数组。通过调用枚举类型的values()方法能够获取该枚举的全部元素构成的数组，方便迭代操作
    @Test
    public void testEnum() {
        ColorEnum color = ColorEnum.BLUE;
        // 编译器报错 incompatible types: se.SeasonEnum cannot be converted to se.ColorEnum
        // color = SeasonEnum.AUTUMN;
        // 获取元素名称
        System.out.println(color);
        // 通过values()迭代操作全部元素
        for(ColorEnum element: ColorEnum.values()) {
            System.out.println(element);
        }
    }
// ----- 输出 -----
BLUE
RED
BLUE

枚举的本质

Java中的枚举本质上还是类和其对象，这通过反编译就能够得到答案

// 源文件SeasonEnum.java
public enum SeasonEnum {
    SPRING, SUMMER, AUTUMN, WINTER;
}
// 反编译SeasonEnum.class
public final class se.SeasonEnum extends java.lang.Enum<se.SeasonEnum> {
  public static final se.SeasonEnum SPRING;
  public static final se.SeasonEnum SUMMER;
  public static final se.SeasonEnum AUTUMN;
  public static final se.SeasonEnum WINTER;
  private static final se.SeasonEnum[] $VALUES;
  public static se.SeasonEnum[] values();
  public static se.SeasonEnum valueOf(java.lang.String);
  private se.SeasonEnum();
  static {};
}

我们仔细瞧一瞧反编译的内容：

  • 该类通过final修饰，因而无法被继承
  • 该类继承自Enum，因此无法继承其他的类
  • 构造方法为private修饰，因此无发构造其对象
  • 所有在源文件中定义的元素都会使用构造一个同名的静态对象

所以，为什么说enum是Java提供的语法糖，因为它是通过编译器进行了类似代码生成的操作，其本质依然是类。

通过了解它是使用类来进行枚举，下面的一些特性就十分好理解了。

枚举和成员

这大概是使用最多的特性了吧，一般使用enum都会在其中包含一些该元素的详细信息，比如实在开发中比较常用的错误枚举，通常就会在定义枚举的时候其错误码以及该错误的具体描述这2个成员。

例如，上述的Season，我希望相应枚举包含对应季节的农历月份以及中文描述。

// Season.java
public enum SeasonEnum {
    SPRING(1, 3, "春"),
    SUMMER(4, 6, "夏"),
    AUTUMN(7, 9, "秋"),
    WINTER(10, 12, "冬");

    private final int fromMonth;
    private final int toMonth;
    private final String desc;

    SeasonEnum(int fromMonth, int toMonth, String desc) {
        this.fromMonth = fromMonth;
        this.toMonth = toMonth;
        this.desc = desc;
    }

    public int getFromMonth() {
        return fromMonth;
    }

    public int getToMonth() {
        return toMonth;
    }

    public String getDesc() {
        return desc;
    }
}

之后，就很明了了：

  • 4个枚举元素相当与使用了全参构造方法进行构造
  • 因为枚举元素是该类的实例，因此可以调用相应的方法获取成员变量

枚举和抽象方法

如果从“枚举是一个类”来看，这个特性确实并不怎么让人吃惊。它的使用场景是当你需要通过枚举的元素不同进行不同的逻辑分支时使用的，相比使用抽象方法更加朴实和笨拙的方式是使用switch...case...。但这样的代码并不优雅，而且也并不利于拓展。

举一个Effective Java中的例子，四则运算。

public enum BasicOperation {
    PLUS("+") {// 加法
        @Override
        public int apply(int left, int right) {
            return left + right;
        }
    },
    MINUS("-") {// 减法
        @Override
        public int apply(int left, int right) {
            return left - right;
        }
    },
    MULTIPLY("*") {// 乘法
        @Override
        public int apply(int left, int right) {
            return left * right;
        }
    },
    DIVIDE("/") {// 除法
        @Override
        public int apply(int left, int right) {
            return left / right;
        }
    };

    private final String symbol;

    Operation(String symbol) {
        this.symbol = symbol;
    }

    public abstract int apply(int left, int right);
}

使用抽象方法能够让枚举在方法层面进行抽象，但它的拓展却是封闭的，由于枚举最终是final类，因此它无法通过继承的方式进行拓展，只能够增加新的枚举元素进行拓展。

与此想对应的，直接编写抽象类，通过该抽象类统筹其实现子类也能达到enum的效果，这在实际的源码中使用的更加频繁。这种情况下，往往抽象类会做以下几件事：

  • 给出相应的工厂方法，通过工厂方法名返回不同的实现

  • 工厂方法内部决定是返回单例对象还是新的实例，使用者无需关心

抽象类统筹实现的好处在于你能够通过抽象类提供的工厂方法，得到所有你可以使用的子类实例，而不需要去了解复杂的继承关系。

枚举和接口

由于enum是默认继承了Enum的，因此无法继承其他类，但依然可以继承接口。

// Operation接口只有一个apply()方法
public enum BasicOperation implements Operation {
    PLUS("+") {
        @Override
        public int apply(int left, int right) {
            return left + right;
        }
    },
    MINUS("-") {
        @Override
        public int apply(int left, int right) {
            return left - right;
        }
    },
    MULTIPLY("*") {
        @Override
        public int apply(int left, int right) {
            return left * right;
        }
    },
    DIVIDE("/") {
        @Override
        public int apply(int left, int right) {
            return left / right;
        }
    };

    private final String symbol;

    BasicOperation(String symbol) {
        this.symbol = symbol;
    }
}

使用接口的好处是你虽然不能拓展BasicOperation，但能够拓展Operation。由于BasicOperation的所有元素是该类的实例之外，也是Operation的实例。因此，当你仅仅需要使用apply()方法时，能够使用Operation作为类型传递枚举对象。这样，你就能定义第2个枚举拓展实现该接口进行拓展了。

我个人感觉，这其实是接口的应用，和枚举关系并不大。如果Operation的拓展能力十分强，那么或许不使用枚举是最好的方式。

枚举和集合

Java提供了2种专门为enum设计的集合类型，EnumSet和EnumMap。其中EnumMap是将enum类型作为key进行存储的Map。

这两种集合对enum进行了特殊的优化，通常的HashSet和HashMap存储内容是通过数组和hashCode()获取散列值确定存储位置的，这是因为冲突的存在无法单纯通过hashCode()直接判断2个元素相等。

然而，enum的元素有着天生独有的哈希值，那就是通过ordinal()获取的序数，序数是绝对不会冲突的哈希值，因为进需要使用位操作和一个数便能够完成通常情况数组才能完成的事。

	// RegularEnumSet
    public boolean contains(Object var1) {
        if (var1 == null) {
            return false;
        } else {
            // 类型判断
            Class var2 = var1.getClass();
            if (var2 != this.elementType && var2.getSuperclass() != this.elementType) {
                return false;
            } else {
                // 通用位操作检查是否存在该元素
                return (this.elements & 1L << ((Enum)var1).ordinal()) != 0L;
            }
        }
    }
    public boolean add(E var1) {
        // 类型检查
        this.typeCheck(var1);
        long var2 = this.elements;
        // 调用ordinal()获取序数并使用位操作存储该序数元素的存在状态
        this.elements |= 1L << var1.ordinal();
        // 通过判断操作前后的数是否相同判断是否添加成功
        return this.elements != var2;
    }

类型检查的原因是contains()的参数类型是Object，因此必须使用类型判断，否则会产生不同的enum类型传入返回true的情况。

    @Test
    public void testEnumConversion() {        
        BasicOperation op = BasicOperation.PLUS;
        Enum e = op;
        EnumSet<SeasonEnum> set = EnumSet.of(SeasonEnum.SPRING, SeasonEnum.SUMMER);
        System.out.println(set.contains(e));
    }
// ----- 输出 -----
false

因此，使用EnumSet和EnumMap不仅仅能够加速查询，并且能够节约内存空间并省去有关负载因子及扩容的大量代码。

总结

Java中的枚举并没有C中那么简单，其本质是类这一特点就决定了它能够胜任一些其他方面的工作，但同时也代表了它比C当中的枚举更加消耗资源。

枚举的虽然能够使用抽象方法进行一定程度的抽象，但无法被继承这一特点也使得它只能够通过修改源码进行拓展。所以，枚举的使用仅在元素可枚举的前提下进行是最佳的。

最后，尽量使用EnumSet和EnumMap。
